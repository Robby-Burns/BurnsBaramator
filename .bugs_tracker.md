## ‚úÖ Resolved Bugs

### BUGS-R04: API Database Connection Failure
**Status:** Resolved
**Priority:** High
**Severity:** Major
**Found:** Feb 2026
**Fixed:** Feb 2026

**Problem:**
`api/index.py` failed because it used SQLite-specific methods (`cursor`, `fetchall`) on the new SQLAlchemy-based `DatabaseManager`. It also hardcoded `jobs.db`.

**Root Cause:**
Refactoring `db/manager.py` to SQLAlchemy broke the API endpoint which was still using the old SQLite pattern.

**Solution:**
Updated `api/index.py` to use `db.engine.connect()` and `text()` queries, and removed the hardcoded DB path to allow env var configuration (Neon/Postgres).

### BUGS-R03: Vercel "Externally Managed Environment" Error
**Status:** Resolved
**Priority:** High
**Severity:** Blocker
**Found:** Feb 2026
**Fixed:** Feb 2026

**Problem:**
Vercel build failed with `error: externally-managed-environment` when running `pip install -r requirements.txt`.

**Solution:**
Removed the explicit `buildCommand` from `vercel.json`. Since `pyproject.toml` is now populated, Vercel's default build process handles dependency installation correctly.

### BUGS-R02: Vercel Build Configuration Mismatch
